### Кейс 3.

Как вы думаете, в чем причина популярности виртуальных серверов последнее время? В чем преимущество виртуальных серверов перед физическими серверами? Какие недостатки есть у виртуальных серверов?

#### Ответ:

1) *Причина популярности виртуальных серверов перед физическими:* 

Заключаются в отсутствие недостатков имеющихся у физических серверов: **Безопасность** – в случае физ.сервера если злоумышленник обнаружит неуязвимость в одной из программ, то это позволит ему получить контроль над всей ОС, в случае решения этой проблемы, переносом некоторых программ на отдельное устройство – мы столкнёмся с **плохой утилизацией ресурсов**; **Ограниченность ресурсов** – если вдруг одна из программ начнёт потреблять большое количество ресурсов вплоть до 100% памяти или процессора, то остальные программы замедлят свою работу, а то и вовсе перестанут работать, при этом покупать сервера под каждый сервис выйдет слишком **дорого**.  

**

А вот виртуальный сервер позволяет изолировать программы друг от друга, не покупая новых серверов и не сталкиваясь с плохой утилизацией ресурсов, при этом решая проблемы с ограниченностью ресурсов и риском взлома всей ОС, так же есть такая возможность как запуск нескольких разных ОС.  

2) *Основные недостатки виртуальных серверов:* 

**Оверхэд** – потребление лишних ресурсов, которые можно было бы и не тратить, за счет эмуляции оборудования и каждой ОС потребляющей дополнительную память, воспроизводящей дополнительные процессы для работы самих ОС.  

**Ограниченное перераспределение ресурсов** – перераспределить ресурсы между каждой ОС можно только на старте, а в дальнейшем, при работе, уже практически будет нельзя что либо поменять, перераспределить.

### Кейс 4 с решением.

Какой из вариантов решения вы выберете для каждой из ситуаций? 

#### Вариант решения:

**A)** Физический сервер, **B)** Виртуальный сервер [2], **C)** контейнеры [3], **D)** SaaS [4], **E)** Прочее (предложите свой вариант).

[1] Если вам **не хватает информации для принятия решения** - укажите, между какими вариантами выбираете и какая информация и как повлияет на ваш выбор.

[2] **Виртуальный сервер** - один из нескольких серверов, запущенных на физическом хосте с помощью гипервизора (например, Ubuntu или Windows Server на гипервизоре VmWare ESXi).

[3] **Контейнеры** - это программное обеспечение, которое "упаковывает" приложение в стандартизованные блоки (контейнеры). Каждый контейнер включает все необходимое для работы приложения: библиотеки, системные инструменты, код и среду исполнения. Контейнеры можно быстро развертывать (запускать) и таким образом масштабировать приложение (например, Docker-контейнеры).

[4] **SaaS** (Software as a service) - программное обеспечение как услуга (например, почтовый сервис Gmail).

#### Ситуации:

* 4.1 Почтовый сервер для небольшой компании из 15 человек без постоянного администратора - **D)** SaaS - не нужно заморачиваться с администрированием;

* 4.2 Сервер для хранения старых резервных копий - **A)** Физический сервер - информация старая, особо ненужная, не страшно при потере из-за сбоев;

* 4.3 Сервер для тестирования приложений в разных окружениях (версиях ОС, версиях языка программирования, и т.д.) - **B)** Виртуальный сервер - прекрасно походит для тестинга в разных окружениях, т.к вирт.сервер располагает возможностью установки и работы разных ОС;

* 4.4 Web-сервер для размещения сайта компании - **A)** Физический сервер - не требовательно, поэтому можно обойтись и физ.сервером;

* 4.5 Старое ПО, работающее только на Windows XP - **C)** контейнеры - удобно для пользователей, чтобы скачивать и устанавливать только необходимые пакеты ПО;

* 4.6 Почтовый сервер в компании из 1500 человек и с собственным ИТ отделом - **D)** SaaS - большая компания располагает приобретением услуг;

* 4.7 Несколько одинаковых серверов, позволяющих запускать и останавливать экземпляры программ по мере необходимости - **C)** контейнеры - очень удобно контролировать запуск и остановку тех или иных программ изолированных друг от друга.

### Кейс 7 с ответами.

#### Нужно ли делать резервное копирование в ситуациях ниже?

* 7.1 Перед внесением изменений в конфигурационный файл на сервере - **Нужно**, для возможности восстановления этого файла в случае совершения ошибок в правке;

* 7.2 Перед удалением с сервера рабочей папки уволенного сотрудника - **Не нужно**, информация уже не нужна и ее можно легко восстановить при необходимости;

* 7.3 Перед применением обновления на работающее ПО на сервере - **Нужно**, во избежание нарушения работы сервера;

* 7.4 Перед изменением документа, лежащего в общем доступе, в режиме "только чтение" - **Нужно**, для возможности возвращения к старой версии по необходимост;

* 7.5 Перед копированием файла из папки проекта на рабочий стол - **Не нужно**, данное действие не меняет содержание файла;

* 7.6 Перед обновлением операционной системы - **Нужно**, в избежание утери временных файлов при перезагрузки.
